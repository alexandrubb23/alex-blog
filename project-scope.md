# Alexandru Barbulescu

## Problem

Currently, when I write an article, I do it in a separate Markdown (MD) file, send it to the AI for improvements, and then manually add it by connecting directly to the MySQL database.

## Solution

Implement an admin-only interface for writing and managing articles, enhanced by AI, without leaving the app.

## Decisions

| Area | Decision |
|---|---|
| Editor | Markdown with live preview |
| Auth | Email/password only (single admin user) via Better Auth (database sessions — no JWT) |
| Article states | Draft → Published |
| Slug | Auto-generated from title, manually editable |
| Hero banner | AI-generated via OpenAI DALL·E from article title/topic, stored on Cloudinary |
| AI classification | Suggests a topic/category while typing (from the existing topics list) |
| AI content improvement | "Improve" button that rewrites the full article body |
| Article deletion | Soft delete (hidden from public, kept in DB) |
| File storage | Cloudinary (images and PDFs) |

## Features

### Authentication
- Email/password login via Better Auth (database sessions — session stored in DB, no JWT)
- Single admin user — no public registration

### Article Editor
- Form fields: Title, Slug (auto-generated from title, editable), Category, Hero Banner, Body
- Markdown editor with live preview for the body
- Hero banner generated by OpenAI DALL·E based on title/topic; uploaded to Cloudinary
- "Improve" button sends the full article body to AI and replaces it with the rewritten version
- AI topic suggestion while typing (matched against the existing topics list)
- Save as Draft or Publish
`
### Admin Dashboard
- Article list with filtering, sorting, and pagination
- Soft delete (sets a `deleted_at` timestamp; hidden from public routes)
- Edit any existing article

### Database Migrations Required
- Add `status` column to `posts` (`draft` | `published`)
- Add `image` column to `posts` (resolves existing TODO in codebase)

## Resume Access Feature

### Problem

Employers need a way to request and download a personalised, up-to-date resume without requiring manual intervention from the admin.

### Solution

A password-gated resume modal that, once unlocked with an admin-generated password, triggers AI-based resume generation (based on LinkedIn-style experience data) and delivers it to the employer as a download and/or via email.

### Decisions

| Area | Decision |
|---|---|
| Entry point | "Resume" button in the NavBar opens a modal with a password field |
| Auth | One-time access password generated by the admin |
| Generation | AI regenerates resume when new experience has been added since the last generated version; otherwise the cached version is reused |
| Delivery | Download button in-browser **and/or** sent to the employer's email address |
| Storage | Generated PDF stored on Cloudinary |
| Current status | API stub always returns an error — full implementation is a future task |

### Features (planned)

- Admin UI to create/revoke access passwords
- LinkedIn-style experience data model (roles, companies, dates, descriptions)
- AI resume generation triggered on password submission when experience has changed
- PDF download link returned in the response
- Optional: email delivery to the employer's address
- Reuse previously generated PDF when experience is unchanged

### Database Migrations Required (planned)

- `experiences` table (LinkedIn-style: role, company, start/end date, description)
- `resume_tokens` table (hashed password, created\_at, used\_at, expires\_at)
- `resume_versions` table (cloudinary\_url, generated\_at, experience\_hash)
